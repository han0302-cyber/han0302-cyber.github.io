> **💡 本篇重點**
> Chatbot 只能回答問題，不能動你的檔案；終端機 AI Agent 是「坐在你電腦前、能直接開檔案做事的數位助理」。它原本是給工程師寫程式用的，但律師拿來管理知識庫，正好。


<!-- more -->
## 一、從「會聊天」到「會做事」

假設你要把二十份判決做成一張爭點比較表。在 ChatGPT 這類聊天框裡，這件事意味著：複製貼上二十次、每貼一份等它讀完，最後再把結果一段一段搬回 Word。因為這種模式有幾個天生的限制——它看不到你電腦裡的檔案；它一次只能處理你貼給它的東西；它給你答案之後，存檔、整理、歸檔還是你自己來。

一個貼切的比喻：**Chatbot 像一位學識淵博、但手腳被綁住的顧問**。什麼都懂，但什麼都不能幫你動手做。

近年 AI 已進入 **Agent（代理）時代**。Agent 與 Chatbot 的差別，不在「更聰明」，而在「能行動」：

| | Chatbot（聊天機器人） | AI Agent（智慧代理） |
| --- | --- | --- |
| 互動方式 | 你問一句、它答一句 | 你交辦一個「任務」 |
| 讀取檔案 | 要你複製貼上 | 自己打開資料夾、逐份讀取 |
| 產出 | 一段文字，要自己複製出來 | 直接建立、修改檔案 |
| 多步驟工作 | 每一步都要你推 | 自己規劃步驟、執行、檢查、交付 |
| 例子 | 「這份判決在講什麼？」 | 「把資料夾裡 20 份判決做成爭點比較表，存成新筆記」 |

## 二、為什麼是「終端機」？

Claude Code（Anthropic）與 Codex（OpenAI）這類工具，最初都以 **CLI（Command Line Interface，指令列介面）** 的形式問世，在**終端機（Terminal）** 裡運作——就是那個黑底白字、要打指令的畫面（兩者如今也都推出桌面版，見第三節的介面比較）。黑底白字的畫面確實容易讓人卻步，先說清楚三件事：

**第一，你不需要會指令。** 打開之後，你跟它的互動就是「用中文打字交辦工作」，跟用 ChatGPT 沒有兩樣。差別只是它在你的電腦裡，看得到你的檔案。

**第二，終端機反而是 Agent 最自然的家。** Agent 要做事，就需要讀寫檔案、執行工具；終端機正是作業系統開放檔案讀寫與工具執行能力的介面。

**第三，它每一步都會先問你。** Claude Code 預設在修改檔案、執行指令前徵求你的同意，你隨時看得到它想做什麼、也隨時可以喊停。主導權在你手上。

## 三、兩大工具速覽

### Claude Code（Anthropic）

- 在終端機輸入 `claude` 啟動，用自然語言（中文完全沒問題）交辦任務。
- 特別會「讀懂一整個資料夾」：把它打開在你的 Obsidian Vault 上，它就能通盤理解你的知識庫結構。
- 支援 **CLAUDE.md**（寫給 AI 的到職須知，見 [CLAUDE.md｜寫給 AI 助理的到職須知](/posts/claude-md-onboarding/)）、**Skills**（可重複使用的 SOP，見 [Skills｜把你的 SOP 變成 AI 的技能](/posts/skills-as-sop/)）與 **MCP**（連接外部資料庫，見 [MCP｜把 AI 接上法規資料庫](/posts/mcp-legal-database/)）。
- 需要 Claude 付費方案或 API 金鑰。

### Codex（OpenAI）

- OpenAI 推出的對應工具，在終端機輸入 `codex` 啟動。
- 概念幾乎相同：讀寫檔案、執行任務、自然語言交辦。
- 對應 CLAUDE.md 的機制叫 **AGENTS.md**，作用一樣：放一份說明檔在資料夾裡，AI 會自動遵守。
- 需要 ChatGPT 付費方案或 API 金鑰。

### 終端機版與桌面版：同一顆引擎、兩種介面

Claude Code 與 Codex 都不只有終端機一種型態，目前各自提供兩種主要介面：

| | 終端機版（CLI） | 桌面版（Desktop app） |
| --- | --- | --- |
| **Claude Code** | 在終端機輸入 `claude` 啟動 | Mac／Windows 應用程式；任務進度與檔案變更用視窗呈現、授權用點選，並可同時管理多個工作階段 |
| **Codex** | 在終端機輸入 `codex` 啟動 | 桌面應用程式，同樣改以視窗操作 |

兩種版本用的是**同一顆 Agent 引擎、同一個訂閱帳號、同一套規則檔**（CLAUDE.md／AGENTS.md），差別只在操作介面：終端機版功能最完整；桌面版把「看進度、按同意、切換任務」變成視窗點選，適合不想開終端機的人。Claude Code 另有網頁版（claude.ai/code）與 IDE 擴充功能，屬於開發者場景，本系列不展開說明。**本系列的操作示範一律以終端機版為基準**；改用桌面版時，提示詞與觀念完全相同。

一個容易混淆的點：**Claude Code 桌面版與下一段的 Cowork 是兩個不同的產品**。桌面版是完整的 Claude Code，只是操作介面不同，規則檔、Skills、MCP 全部照用；Cowork 則是 Claude 桌面應用程式內建、面向一般工作者的代理功能，設計成「選資料夾、交辦、取件」的簡化流程。

### 同場加映：Claude Cowork——不想碰終端機的選項

如果你偏好圖形介面的操作方式，Anthropic 也準備了一個容易上手的選項：2026 年 1 月推出的 **Claude Cowork**，官方定位非常傳神——「**Claude Code without the code**」。同一顆 Agent 引擎（讀寫檔案、多步驟任務、自主規劃執行），包進 Claude 桌面應用程式（macOS / Windows）熟悉的聊天視窗，2026 年 7 月起並陸續擴展到網頁版與行動版。用法只有一句話：**選一個資料夾，然後用中文說話**。沒有指令、沒有黑畫面，它就能讀取、整理、產出檔案——請它把「請款」資料夾裡的發票整理成 Excel、把二十份判決做成比較表、把英文契約翻成中日對照版，都是它的守備範圍。訂閱與 Claude Code 共用同一個付費方案（免費帳號不可用）。

同一顆引擎、兩種包裝，怎麼選？關鍵在工作型態：**Cowork 適合「一次性交付」的檔案任務；Obsidian ＋ Claude Code 適合「持續共筆」的知識工作。** Cowork 的互動本質是「交辦→交付」：每個工作階段指定一個資料夾、存取範圍以該資料夾為限，檔案做好後你要到 Finder／檔案總管找出來、再用其他軟體打開來改。而本系列的主場景——知識庫的日常維護——是「人改一段、AI 改一段」的來回節奏：左邊 Obsidian 開著筆記、右邊終端機下指令，AI 改完，變動立刻呈現眼前，你隨時接手編輯再丟回去加工；中途要用其他資料夾的資料，貼個路徑就能延伸。這是本系列以 Claude Code 為主線的原因。

給律師的建議路線：

- **想從圖形介面開始**：從 Cowork 起步——它是體驗「AI 會做事」最容易上手的入口；要向所內同仁推廣 Agent，它也是合適的起點。
- **要走本系列完整路線（CLAUDE.md、Skills、MCP）**：主力用 Claude Code，這些進階機制在終端機環境最完整。
- **兩者並用完全沒問題**：同一個訂閱帳號，日常文件雜務丟 Cowork，知識庫編纂與開發工作用 Claude Code。
- 另一條「圖形介面用 Agent」的路，是 [Obsidian 進階｜外掛與 Canvas 白板](/posts/obsidian-plugins-canvas/) 介紹過的社群外掛 **Claudian**：把 Claude Code 直接嵌進 Obsidian 介面，與 Cowork 異曲同工。

> **⚠️ Cowork 的資安預告**
> Cowork 的權限設計比 Claude Code 流暢（較少逐步確認），這是雙面刃：好用，但也代表你**指定資料夾前要先想清楚裡面有什麼**——給它讀的資料夾，內容視同交給雲端模型處理；客戶機密資料必須先確認方案等級（商用方案與個人 Pro 的條款不同）。第一次使用，拿公開資料或去識別化資料練習。詳見 [資安與倫理｜律師使用 AI Agent 的紅線](/posts/security-ethics-red-lines/)。

**本系列以 Claude Code 為主要示範工具**（生態較完整，且 MCP、Skills 等機制對法律工作流的支援較成熟），但所有觀念——Vault 設計、到職須知、編纂心法——在 Codex 上同樣適用，只要把 `CLAUDE.md` 換成 `AGENTS.md` 即可。學觀念，不要綁工具。

更根本的保障是：把規則（CLAUDE.md）、技能（Skills）、知識（Vault 筆記）通通存成自己資料夾裡的純文字檔——**真正累積下來的資產，是這些檔案，而不是任何一套特定工具**；哪天要更換 Agent，整包資料夾搬走即可。

## 四、當 Agent 遇上 Obsidian：一加一大於二

單獨看，Obsidian 是一個「需要人工整理」的筆記軟體；單獨看，Claude Code 是一個「給工程師寫程式」的工具。但把兩者放在一起：

> **Obsidian Vault 就是一個資料夾，Claude Code 最擅長的就是讀寫資料夾。**

於是你得到一個這樣的組合：

1. **你負責丟材料**：判決、法規、論文筆記、研究心得，丟進 Vault。
2. **AI 負責編纂**：讀取新材料、寫成結構化筆記、補上 wikilink、更新主題頁（MOC）、標出見解衝突。
3. **Obsidian 負責呈現**：漂亮的閱讀介面、雙向連結、知識圖譜，隨時檢索。

實際的畫面是這樣的——以主案例 2026-015 的研究主題「工期展延」為例，你在終端機打一句：

```
請讀取「inbox」資料夾裡我今天丟進去的三份判決筆記，
每一份整理出：判決字號、案由、爭點、法院見解，
然後在「工期展延」主題頁底下更新實務見解清單，
相關筆記之間用 [[wikilink]] 連結。
```

然後你就看著它自己打開資料夾、一份一份讀、動手寫筆記、補連結。你從頭到尾只打了這一段中文。

> **💡 提示詞的四要素**
> 跟任何 AI 溝通（不只 Agent），把四樣東西講清楚，產出品質立刻不同——**角色**（「你是一位專精營建工程爭議的資深律師」）、**任務**（動詞明確：分析、比較、翻譯）、**脈絡**（案件背景、你在意什麼——律師最常省略、卻最關鍵的一塊）、**格式**（要表格、條列，還是限制字數）。上面那段交辦任務的中文，正是四要素的實作。本系列之後所有的提示詞範本，都可以用這四個要素來檢視與修改。
>
> 在 Agent 環境裡，「角色」這一項往往可以省下來：[CLAUDE.md｜寫給 AI 助理的到職須知](/posts/claude-md-onboarding/) 會教你把身分、專業領域與行文要求一次寫進 CLAUDE.md，AI 每次啟動都自動帶入，不必每則提示詞重講。之後真正需要逐次交代的，主要是任務、脈絡與格式；只有臨時要換視角（例如「改從對造律師的立場檢視這份契約」）時，才需要在提示詞裡另外指定角色。

## 五、不需要會寫程式：來自實際數據的觀察

Anthropic 曾分析約四十萬次 Claude Code 的實際使用，幾個發現對律師特別有參考價值：

1. **成功率隨「領域專業度」提升**，而不是隨「程式能力」提升。你對法律問題理解得越深，AI 產出的品質越高。
2. 分工非常固定：**人決定「做什麼」，AI 決定「怎麼做」。** 這與資深律師和助理的分工相同。
3. 非軟體背景的使用者成長迅速，法律工作者也在其中。

換句話說：**你的法律專業，才是把這套工具用好的關鍵。** 判斷「哪個爭點重要」「哪則判決有代表性」「這段分析對不對」——這些判斷 AI 做不到。

這個轉變有個名字，Anthropic 主管稱之為 **Vibe Working**：工作者不再逐字逐句親手操作，而是**描述想要的結果，AI 完成大部分執行細節，人負責品質把關**。

對律師的意義：大量重複性的文字勞動，未來由 AI 分擔的比重會越來越高；律師的價值將更集中於四件 AI 做不到的事——**判斷力、經驗、客戶關係、策略思維**。角色從「執行者」變成「品質守門人」，這個轉變貫穿本系列每一個工作流的設計（人在頭尾、AI 在中間），到 [資安與倫理｜律師使用 AI Agent 的紅線](/posts/security-ethics-red-lines/) 會做最後的收束。

## 六、放眼生態系：OpenClaw、Hermes 與本系列路線的差異

2026 年的個人 AI Agent 已經是一個熱鬧的生態系。除了 Claude Code / Codex，你大概也聽過兩個廣受矚目的開源專案：**OpenClaw**（Peter Steinberger 開發）與 **Hermes Agent**（Nous Research 開發）——兩者概念相近，都是自架的「常駐通訊助理」：把 AI 模型接上 WhatsApp、Telegram 等通訊軟體，24 小時待命，能在你的電腦或伺服器上執行指令、讀寫檔案、收發郵件。跟「Obsidian × Claude Code」路線相比，差異在三個層面：

| | **OpenClaw / Hermes** | **Obsidian × Claude Code（本系列）** |
| --- | --- | --- |
| 型態 | **常駐通訊助理**：架在伺服器上 24 小時待命，用傳訊息的方式差遣 | **檔案知識庫協作**：要用時打開，在你的筆記與卷宗上工作 |
| 工作的中心 | 「對話與任務」——幫你跑腿、查資料、發訊息 | 「檔案與知識」——編纂筆記、整理判決、起草文書，成果沉澱為知識庫 |
| 門檻與風險 | 需要自架與維運；**把終端機、瀏覽器、信箱的權限交給一個常駐機器人**，權限範圍大、出事影響也大 | 本機執行、逐步授權、每個動作可審查；工作範圍限縮在你指定的資料夾 |

有趣的是，OpenClaw 與 Hermes 的「記憶」與「技能」也都用純文字 Markdown 檔儲存——這印證了本系列的核心押注：**純文字檔案是 AI 時代知識的通用貨幣。**

本系列不選它們作主線，主要是**工作型態不合**：這類常駐助理以聊天訊息為互動中心，擅長「交辦一件事、回報一個結果」的零散任務，但產出散落在對話串裡，不容易沉澱成有結構的知識庫。而本系列要建立的是「材料進 Vault → AI 編纂 → 成果回流」的循環——判決筆記、爭點 WIKI、書狀底稿，工作與成果都以檔案為中心，Obsidian × Claude Code 直接在這些檔案上工作，每一次產出都留在知識庫裡繼續累積。此外，自架與維運本身是一門功課，常駐機器人接上通訊軟體與信箱後的權限管理，也是需要另行評估的課題。行有餘力，再嘗試這些新興工具也不遲。（以上是就律師工作型態所做的適用性評估，並非對這兩個專案本身的評價。）

## 七、小結

- Agent 與 Chatbot 的本質差異：能不能「動手做事」。
- 終端機不可怕：你用中文交辦，它徵求同意後執行，主導權在你。
- 兩套工具都有終端機版（CLI）與桌面版，引擎與帳號相同；本系列示範以終端機版為基準。
- Claude Code / Codex + Obsidian = AI 管理員 + 地端知識庫。
- 偏好圖形介面？Claude Cowork 是「選資料夾＋說中文」的入門起點；但「持續共筆」的知識庫工作，主線仍是 Claude Code。
- 下一篇就實際動手，把 Claude Code 裝進知識庫。
