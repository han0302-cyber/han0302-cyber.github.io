> **💡 本篇重點**
> Obsidian 是地端的，但 AI Agent 讀取的內容會送到模型公司的伺服器——所以「Vault 裡放什麼、給 AI 讀什麼」必須分級管理。付費不等於安全，方案類型才是關鍵；而引用查證與最終責任，永遠在律師身上。


<!-- more -->
這是全系列最重要的一篇。前面各篇談的是「怎麼用」，這一篇談的是「能不能用、用到哪裡」。

## 一、先講清楚資料流：地端知識庫的雲端縫隙

本系列反覆強調 Obsidian 的地端優勢，但必須誠實面對一個關鍵事實：

> **筆記的「存放」在你的電腦；但當 Claude Code / Codex 讀取筆記執行任務時，讀到的內容會作為提示傳送到 Anthropic / OpenAI 的伺服器運算。**

所以「知識庫是地端的」這句話只對了一半。完整的圖像是：

| 環節 | 資料在哪裡 |
| --- | --- |
| Obsidian 筆記存放 | ✅ 你的電腦（除非 Vault 放在同步資料夾） |
| 你自己閱讀、編輯 | ✅ 你的電腦 |
| AI Agent 讀取檔案執行任務 | ⚠️ 讀取的內容上傳至模型公司伺服器 |
| AI 產出寫回筆記 | ✅ 你的電腦 |

結論：**給 AI 讀什麼，就等於把什麼交給了模型公司。** 這不必然違規——關鍵在你用的是哪種方案、對方是什麼法律地位、內容是什麼等級。往下看。

## 二、付費 ≠ 安全：方案類型才是關鍵

常見的理解是「我付月費了，所以應該安全」，但這並不精確——消費者方案再貴，本質仍是消費者方案。律師必須區分的是**你與廠商之間的法律關係**：

| 途徑 | 資料用於訓練？ | 法律關係 | 客戶資料可否 |
| --- | --- | --- | --- |
| 免費版網頁 | 預設可能用於訓練 | 消費者條款 | ❌ |
| 付費個人版（如 Claude Pro／Max、ChatGPT Plus／Pro） | 同上（見下方說明） | 消費者條款 | ❌ 原則不可 |
| 商用訂閱方案（Claude Team／Enterprise、ChatGPT Business／Enterprise 等） | 預設不訓練 | 有 DPA（資料處理協議） | ⚠️ 依所內政策 |
| API | 預設不訓練 | 商業條款 | ⚠️ 依所內政策 |
| 地端模型（本機運行） | 資料不出你的電腦 | — | ✅ 技術上最安全 |

方案名稱與適用條款以各家官方頁為準。本系列用到的兩個 Agent，分級邏輯其實一致：

- **Claude（Anthropic）**：個人方案為 Pro／Max（消費者條款）；商用訂閱分 **Team**（5 人即可成團，適合中小型事務所）與 **Enterprise** 兩級，適用商業條款、可簽 DPA。Claude Code 以哪個方案登入，就適用哪個方案的條款。
- **Codex（OpenAI）**：可透過 ChatGPT 訂閱或 API 金鑰取用，Codex CLI 兩種登入方式都支援。其中 **Plus、Pro 屬個人方案**（消費者條款），**Business、Enterprise 屬商用方案**（適用商業條款、可簽 DPA）。商用方案與 API 預設不將資料用於訓練；個人與免費方案則預設可能用於訓練，雖可在設定中關閉，但本質仍是消費者條款、沒有 DPA。

兩條原則對兩家都成立：

- **用哪個方案的額度登入，就適用哪個方案的條款**——以 Pro／Plus 等個人訂閱登入，適用消費者條款；改接 API 金鑰，適用商業條款。這直接決定你能餵它什麼資料。
- **用任何工具前，先查它當下的資料使用政策**——各家的方案名稱與訓練政策都還在變動，別憑印象；查完把結論寫進所內規範。

## 三、律師的三重合規框架

為什麼這是「法律問題」而不只是「商業考量」：

**1. 保密義務**。律師法第 36 條課予律師保密義務；企業法務亦負有僱傭契約與公司內規上的保密義務。把客戶案件資料輸入「預設用於訓練」的消費者版 AI，與保密義務的相容性有重大疑慮。

**2. 個資法的委託處理**。將含個資的資料交由 AI 服務處理，屬於委託處理個人資料，依個人資料保護法施行細則第 8 條，委託機關應對受託者為適當之監督。消費者方案沒有 DPA、沒有稽核機制，難以主張已盡監督義務；商用方案與 API 的商業條款才有對應的架構。

**3. 涉外業務的 GDPR**。服務歐盟客戶時還有控制者／處理者的定性問題：廠商若將你的輸入用於訓練自己的模型，它就不只是處理者——這是合規上的重大風險點。

## 四、三層分流：可操作的決策規則

把以上整理成一個可以貼在牆上的規則——**資料越敏感，工具越要往「自己掌控」那一端靠**：

| 層級 | 資料性質 | 可用工具 | Vault 對應 |
| --- | --- | --- | --- |
| **第一層** | 非敏感：公開判決、法規、文獻、教學、去識別化素材 | 個人訂閱即可 | `20_AREAS`、`30_RESOURCES` 的研究軌內容 ✅ 放心給 AI |
| **第二層** | 一般案件工作：含當事人資訊的卷證、書狀 | 事務所的商用方案／API | `10_MATTERS` 案件軌 ⚠️ 依所內政策，且盡量去識別化 |
| **第三層** | 最敏感：偵查中案件、重大交易、高度隱私 | 地端模型，或不用 AI | 考慮**根本不放進 Vault**，或放在 AI 禁止讀取的資料夾 |

以本系列的案例線為例：你歸檔的公開判決屬第一層；2026-015 案的卷證與書狀底稿屬第二層。

落實到 Obsidian ＋ Agent 的四個實際手段：

1. **Vault 分區**：在 CLAUDE.md 白紙黑字寫「禁止讀取 `10_MATTERS/_機密/` 內任何檔案」；更嚴格的做法是敏感案件用**獨立的 Vault**，只在商用方案／地端模型環境開啟。
2. **入庫即去識別化**：研究軌筆記從建立時就不含當事人資訊（[檔案整理方法論｜案件與研究領域的雙軌知識系統](/2026/07/07/file-organization-methodology/) 的「知識萃取」正是這個原則的制度化）。
3. **地端模型選項**：Ollama 等工具可在自己電腦運行開源模型，搭配本機轉檔工具（如 markitdown），可組出「資料完全不出機」的處理管線。能力不如雲端旗艦模型，但對第三層資料，這是「能用 AI」與「不能用 AI」的差別。
4. **技術性護欄（deny 規則與 hooks）**：CLAUDE.md 的禁令是「所規」，AI 極高機率遵守但非強制。要技術層級的強制，主力是 Claude Code 設定檔（settings.json）的**權限拒絕規則（deny）**：一是把「讀取 `_機密/` 路徑」寫進 deny 清單，工具層一律擋下、連問都不問；二是把「無論如何都不准執行」的高危指令寫死——例如遞迴永久刪除、強制覆寫版本歷史、變更系統權限——就像事務所的印鑑章有專人保管、任何人都不能單獨用印，最危險的動作本來就不該進入日常授權的範圍。另有 **hooks（掛鉤）** 機制，可在工具執行前後掛上自訂檢查。對 AI 說「請幫我在 settings.json 建立禁讀路徑與高危指令的拒絕清單」，讓它自己把柵欄立好。

另外三個容易忽略的縫隙：

- **同步服務**：Vault 放在 OneDrive / iCloud，資料就在雲端，屬於另一條委外關係；
- **Obsidian 社群外掛與 MCP 伺服器**：第三方程式碼，可能讀取 Vault 內容，只裝必要且可信的；
- **Shadow IT**：所內沒有合規方案，同仁便可能轉而使用個人帳號——管理者的正解是「提供合規替代方案＋教育訓練」，不是一紙禁令。

## 五、倫理面：AI 時代不變的三件事

**1. 查證，沒有例外。** 國際上已有多起律師因引用 AI 杜撰判例遭法院制裁的案例——最著名的是 2023 年美國 Mata v. Avianca 案，紐約律師引用 ChatGPT 虛構的判決先例，遭法院處罰；其後各國仍不斷出現同類事件，涉事者不乏具規模的事務所。教訓很簡單：有規範不夠，還要被遵守；查證沒有例外。 本系列的制度化手段：CLAUDE.md 的「引用標註待核對」規則＋ [MCP｜把 AI 接上法規資料庫](/2026/07/07/mcp-legal-database/) 的資料庫初核＋你的最終人工複核，三道防線。

**2. 責任不可移轉。** AI 產出的書狀、意見書、筆記，署名的是你。「這是 AI 寫錯的」在法庭上、在客戶面前、在懲戒程序中，都不成立。這也是為什麼本系列所有工作流的設計都是「人在頭尾」：你定方向，你做定稿。

**3. 大腦要保持參與。** MIT 媒體實驗室 2025 年的腦波研究（樣本不大、屬初步發現，但值得警惕）顯示，長期讓 AI 代思考的受試者出現「認知負債」——即使停用 AI，認知活躍度也難以回復。對策不是拒用，而是順序：**你先想、AI 補充；你先判斷、AI 檢查。** 讓 AI 跟你「一起想」，不要「替你想」。

本系列從第一篇就反覆劃出的那條界線，在這裡做最後收束：**AI 可以替你整理，但思考與理解，永遠是律師自己的工作。** 摘要、索引、爭點 WIKI，都只是把「值得讀的東西」送到你面前的輸送帶；真正把知識變成專業判斷的，仍然是你逐字讀過、親自想通的那個過程。這套系統的全部承諾是替你省下整理的勞動；如果連閱讀與思考也一併省掉，那不是效率，是**大腦外包**——而外包出去的，恰恰是客戶付費購買、AI 又無法替你承擔責任的那個部分。知識庫再完整，沒有經過你大腦的知識，終究不是你的。

對年輕律師而言，基本功階段或許值得刻意慢學——自己查法條、自己讀判決，AI 用來檢查成果，而不是取代訓練。

對資深律師而言，把 know-how 寫成 Skills 與知識庫（見 [Skills｜把你的 SOP 變成 AI 的技能](/2026/07/07/skills-as-sop/) 的傳承一節），它不會取代你，反而放大你——並讓專業在事務所留下制度性的傳承。

## 六、行動檢查表

**今天就做：**
- [ ] 檢查你的 Claude / ChatGPT 帳號的資料訓練設定
- [ ] 確認你的 Vault 存放位置（本機？同步資料夾？）與內容等級是否相符
- [ ] 在 CLAUDE.md 寫入禁讀資料夾與引用查證規則

**制度化：**
- [ ] 依三層分流盤點你的日常資料，寫成自己（或所內）的一頁式規範
- [ ] 敏感案件另立 Vault 或資料夾隔離
- [ ] 建立「入庫即去識別化」與「知識萃取」的習慣
- [ ] 管理者：提供合規方案＋教育訓練，取代單純禁止

## 結語：AI 給你的是時間

這個系列介紹了很多技術，最後只想收在一件事上：AI 接手的是重複的資訊勞動，它給你的是時間。把省下來的時間，投入無法被取代的事。
