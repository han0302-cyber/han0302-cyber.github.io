> **💡 本篇重點**
> Markdown 是「用符號表達格式」的純文字寫法——`#` 是標題、`-` 是條列、`**` 是粗體。律師夠用的語法只有十個，十分鐘學完，而且它正是 AI 的母語。


<!-- more -->
## 一、為什麼律師要學 Markdown

上一篇的練習停在一個現場：從司法院裁判書系統複製一則判決、貼進 Vault，得到的是一整片沒有格式的文字——主文、爭點、理由全部糊在一起，幾萬字讀起來比紙本卷宗還吃力。Markdown 就是解這一題的工具：用幾個符號，讓這片文字長出結構。

除了這個眼前的痛點，還有三個理由：

**1. 它是 AI 的母語。** ChatGPT、Claude 的回答原生就是 Markdown 格式；Claude Code 讀寫的檔案是 Markdown；你的 Obsidian 筆記也是 Markdown。學會它，你和 AI 之間就沒有翻譯損耗——你甚至可以在提示詞裡直接指定「第一層標題用 `##`、重點用粗體」。

**2. 格式與內容分離，專注寫作。** Word 提供大量格式選項，寫作時容易分心去調字型、行距；Markdown 沒有這些選項，格式就是那幾個符號，寫作時只剩內容本身。而需要交件時，AI 可以一句話把 Markdown 轉成排版好的 Word（「請把這份筆記轉成 docx，套用事務所書狀格式」）。

**3. 純文字永不過時。** `.md` 檔本質是 `.txt`，三十年後任何電腦都打得開。對職涯長達數十年的律師，這是格式壽命的保證。

## 二、十個夠用的語法

以下就是全部了。左邊是你打的字，右邊是 Obsidian 顯示的效果。

### 1. 標題：`#`

```markdown
# 判決字號（大標題）
## 壹、事實摘要（第二層）
### 一、原告主張（第三層）
```

`#` 越多層級越低，最多六層。**標題是 Markdown 最重要的語法**：它讓 AI 與 Obsidian 的大綱面板都能理解文件結構——書狀習慣的「壹、一、（一）」層次，直接對應 `##`、`###`、`####`。

### 2. 粗體與強調：`**`

```markdown
乙公司**自始未曾**就追加工項另行給付報酬。
```

顯示為：乙公司**自始未曾**就追加工項另行給付報酬。（斜體 `*字*` 在中文排版效果不佳，可以不用。）

### 3. 條列：`-`

```markdown
- 原證一：工程承攬契約
- 原證二：變更設計會議紀錄
    - 縮排兩格（或按 Tab）就是子項目
```

### 4. 編號清單：`1.`

```markdown
1. 收受起訴狀
2. 提出答辯狀
3. 言詞辯論
```

只要開頭是 `1.`，後面的編號會自動接續，插入新項目也不用手動改號——寫過長書狀的人都懂這有多珍貴。

### 5. 待辦核取方塊：`- [ ]`

```markdown
- [ ] 8/1 前提出證人詰問問題清單
- [x] 已閱卷（完成的項目打 x）
```

在 Obsidian 中顯示為可勾選的方塊，案件待辦清單直接可用。

### 6. 引用區塊：`>`

```markdown
> 最高法院110年度台上字第XXXX號：「……審酌一切情狀，
> 認系爭違約金約定過高，依民法第 252 條減至相當數額……」（待核對）
```

引用判決原文、法條條文的標準寫法，視覺上自動縮排加線，跟自己的分析明確區隔。

### 7. 表格：`|`

```markdown
| 期日 | 程序 | 待辦 |
| --- | --- | --- |
| 8/12 | 言詞辯論 | 詰問清單 |
| 9/30 | 宣判 | — |
```

第二行的 `| --- |` 是表頭分隔線，必須有。手打表格稍麻煩，實務上通常是叫 AI 產表格，你只要看得懂、能小改就行。

### 8. 分隔線：`---`

單獨一行三個減號，就是一條水平分隔線。典型用法是在同一則筆記裡分隔「摘要區」與「全文區」——[實戰｜打造判決知識庫：從裁判書到 WIKI 索引](/2026/07/07/judgment-knowledge-base/) 整理裁判書時正是這樣用。

### 9. 連結

```markdown
[[最高法院110年度台上字第XXXX號]]        ← 內部連結（wikilink）
[司法院裁判書系統](https://judgment.judicial.gov.tw/)  ← 外部網址
```

wikilink 連筆記，`[文字](網址)` 連外部網頁。

### 10. 程式碼區塊：三個反引號 ```` ``` ````

````markdown
```
這裡面的文字會原樣顯示，不做任何排版
```
````

律師用它來放「要原樣保存的文字」：提示詞範本、法條原文、要貼給 AI 的指令。本系列文章裡所有灰底區塊都是它。

## 三、Obsidian 特有的四個擴充語法

### Callout 色塊：`> [!tip]`

Obsidian 把引用區塊擴充成彩色提示框：

```markdown
> [!tip] 重點摘要
> 綠色區塊，適合放結論與摘要。

> [!question] 爭點整理
> 黃色區塊，適合放問題與爭點。

> [!note] 法院見解
> 藍色區塊，適合放見解與說明。

> [!warning] 注意
> 紅黃色警示，適合放期限與風險。
```

色塊讓不同類型的資訊一眼就能區分。本系列後面會大量用到它：[實戰｜打造判決知識庫：從裁判書到 WIKI 索引](/2026/07/07/judgment-knowledge-base/) 會教你讓 AI 在每份裁判書的開頭，自動產生「重點摘要／爭點整理／法院見解」三個色塊（本系列稱之為「三色摘要」）——這裡先把語法記起來即可。

### 螢光標記：`==文字==`

```markdown
本件關鍵在於==乙公司於 2024 年 3 月變更設計==之事實。
```

用兩組等號夾住的文字會以螢光底色顯示，適合標記判決摘要中的關鍵事實。需要自訂顏色時，可改用 HTML 的 `<mark>` 標籤。

### 嵌入預覽：`![[ ]]`（PDF 可指定頁碼）

在 wikilink 前面加一個驚嘆號，連結就升級成「嵌入」——檔案內容直接顯示在筆記裡。對律師最有感的用法是**把電子卷證、判決或對造書狀的 PDF 直接嵌進筆記**：

```markdown
![[原證三_工程承攬契約.pdf]]              ← 嵌入 PDF，筆記內直接翻閱
![[對造答辯二狀.pdf#page=7]]              ← 預覽直接停在第 7 頁
![[監造月報_0512.pdf#page=12&height=450]] ← 加 height 限制預覽視窗高度
![[證物照片.png]]                          ← 圖片同理
![[最高法院110年度台上字第XXXX號]]          ← 另一則筆記也能整則嵌入
```

`#page=` 是實戰精華：分析筆記寫到「對造就變更設計次數之主張前後矛盾」時，直接嵌入對造書狀 PDF 並停在關鍵頁——開庭前複習、與同事討論時，證據原文就在論點旁邊，不必重新翻卷。電子卷證動輒數百頁，一則爭點筆記嵌入三、五個「停在關鍵頁」的預覽，就是一份自帶卷證的攻防分析。

（提醒：PDF 檔案本身必須放在 Vault 資料夾內才能嵌入；嵌入只是「顯示」，原檔完全不會被修改。）

### 屬性區（frontmatter）：檔案開頭的 `---` 夾層

```markdown
---
date: 2026-07-07
court: 最高法院
case_type: 民事
tags:
  - 營建工程
  - 工期展延
---
```

放在**檔案最開頭**、用兩行 `---` 夾起來的區域，是這則筆記的結構化資料（戶籍欄）。Obsidian 會把它顯示成表單樣式，AI 編纂時也靠它做分類與檢索。

> **⚠️ 標籤有格式限制，違規會被畫刪除線**
> Obsidian 的標籤只允許文字、數字、連字號（`-`）、底線（`_`）與斜線（`/`），**不能包含空格與句點**。如果你在屬性面板看到某個標籤被畫上刪除線，不是內容壞了，而是 Obsidian 在告訴你「這個標籤格式無效、無法點擊與聚合」。常見的無效寫法與修正：`Claude Code` → `Claude-Code`、`AI Agent` → `AI-Agent`、`個資法施行細則第8條` ✅ 沒問題（數字可以）、`民法§184` ❌（特殊符號不行）。改成合法格式後，刪除線就會消失，標籤恢復可點擊、可搜尋。

## 四、律師的完整範例：一則判決筆記

把以上語法組合起來，一則實戰筆記長這樣：

```markdown
---
date: 2026-07-07
court: 最高法院
case: 110年度台上字第XXXX號
tags: [營建工程, 違約金酌減]
---

# 最高法院110年度台上字第XXXX號民事判決

> [!tip] 重點摘要
> 工程遲延違約金酌減案。法院審酌一切情狀，認系爭違約金約定**過高**，
> 依民法第 252 條減至相當之數額（待核對）。

## 爭點

1. 工期遲延是否可歸責於承攬人
2. 遲延違約金應否依民法第 252 條酌減

## 我的評析

- 與 [[臺灣高等法院110年度建上字第XX號]] 酌減幅度不同，本則審酌因素較完整
- 可用於 [[2026-015_工程款案]] 的備位主張

## 待辦

- [ ] 核對判決字號與引用段落
```

## 五、練習方式：讓 AI 當你的 Markdown 家教

不用背語法。打開 Claude（網頁版即可）說：

```
我在學 Markdown。請把以下這段文字轉成結構良好的 Markdown 筆記，
並在每個語法旁用註解告訴我你用了什麼、為什麼：
（貼上任何一段你的工作筆記）
```

來回三次，語法就內化了。之後在 Obsidian 寫筆記，格式不會的地方就問 AI。

## 小結

- 十個語法：標題、粗體、條列、編號、待辦、引用、表格、分隔線、連結、程式碼區塊。
- 四個 Obsidian 擴充：callout 色塊、螢光標記、嵌入預覽（PDF 可指定頁碼）、frontmatter 屬性。
- 心法：結構讓標題扛、重點讓粗體扛、來源讓引用扛；格式轉檔交給 AI。
